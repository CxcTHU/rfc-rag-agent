# pytest 与 TestClient

所属阶段：[[阶段 0 - FastAPI 工程底座]]
所属分类：[[测试与验证]]
相关位置：`tests/test_health.py`

## 它解决什么问题

手动访问接口只能证明当前这一次可用，不能保证后续改代码时不破坏已有行为。自动化测试用于把关键行为固定下来。

## 在本项目中怎么用

阶段 0 的测试会验证：

- `/health` 返回 HTTP 200。
- JSON 内容等于预期结构。

测试命令：

```powershell
python -m pytest
```

## 为什么这样设计

`TestClient` 可以在不启动真实端口的情况下模拟 HTTP 请求，测试速度快，适合做接口回归测试。

## 面试可能怎么问

为什么 `/health` 这么简单也要写测试？

## 你应该怎么回答

因为它验证的不只是一个返回值，还验证应用能导入、路由已注册、响应模型能序列化。后续每次改配置、路由或应用入口，测试都能快速发现是否破坏了基础服务。
