# 题录元数据语料库

所属阶段：[[阶段 1 - 本地资料导入与关键词检索]]
所属分类：[[数据工程]]
相关位置：`app/services/source_collection.py`、`scripts/collect_metadata_corpus.py`

## 它解决什么问题

论文全文下载慢，而且经常受版权、机构账号、验证码和网站反爬限制影响。题录元数据语料库先收集题名、作者、年份、期刊、摘要、关键词、DOI 和链接，让系统先拥有更大的检索覆盖面。

## 在本项目中怎么用

`scripts/collect_metadata_corpus.py` 从 OpenAlex、Crossref、Semantic Scholar 或 CNKI/Google Scholar 辅助导出文件中收集题录，写入 `data/metadata/rfc_papers_metadata.csv` 和 `data/metadata/rfc_papers_metadata.jsonl`，再生成 Markdown 卡片导入 `documents/chunks`。

## 新词解释

- 题录：一篇论文的登记信息，不是全文。
- 元数据：描述资料的数据，例如题名、作者、年份、来源、DOI。
- metadata corpus：由元数据和摘要组成的语料库。
- JSONL：一行一个 JSON 记录，适合存放大量题录。

## 为什么这样设计

阶段 1 的目标是先让资料能导入、能检索、能追踪来源。题录卡片可以复用现有 Markdown parser、cleaner、splitter、repository 和 keyword search，不需要马上设计复杂的论文元数据表。

## 面试可能怎么问

问：为什么不直接爬 Google Scholar 和 CNKI？

答：Google Scholar 没有官方公开批量 API，直接抓页面容易触发验证码；CNKI 有机构授权边界。项目主链路使用 OpenAlex、Crossref、Semantic Scholar 这类可批量访问的学术元数据接口，同时支持 CNKI 或 Google Scholar 辅助工具导出的 CSV/RIS 文件导入。这样更稳定，也更容易解释数据来源。
