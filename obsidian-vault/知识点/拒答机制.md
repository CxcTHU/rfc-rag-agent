# 拒答机制

所属阶段：[[阶段 3 - 引用式问答]]
所属分类：[[RAG 链路]]
相关位置：`app/services/generation/answer_service.py`

## 它解决什么问题

RAG 系统最重要的可靠性要求之一是“不知道就说不知道”。如果资料库没有依据，模型不能凭常识或想象回答工程问题。

## 在本项目中怎么用

`CitationAnswerService` 在这些情况下拒答：

- 检索结果为空。
- 检索结果低于 `min_score`。
- prompt builder 无法构造有效上下文。

拒答答案统一为：

```text
当前资料库中没有找到足够可靠的依据。
```

## 新词解释

- refused：响应里的布尔字段，表示本次是否拒答。
- refusal_reason：拒答原因，方便排查。
- min_score：最低相关性阈值，低于它的结果会被过滤。
- 硬编：模型在没有资料依据时编造答案。

## 为什么这样设计

不能只靠 prompt 告诉模型“不要乱答”。工程层也要先判断有没有可靠资料。这样即使模型能力不稳定，也能在资料不足时给出明确拒答。

## 面试可能怎么问

问：RAG 系统怎么处理知识库没有答案的问题？

答：我在 service 层做拒答判断。没有检索结果或低于分数阈值时，不让模型自由发挥，而是返回统一拒答文案和 refused 状态。

## 你应该怎么回答

拒答不是失败，而是可靠性的体现。尤其是水利工程类问题，系统不能替代规范审查和专家判断；资料库没有足够依据时，应该明确告诉用户没有可靠资料。
