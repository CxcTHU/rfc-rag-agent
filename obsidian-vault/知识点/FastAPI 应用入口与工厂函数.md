# FastAPI 应用入口与工厂函数

所属阶段：[[阶段 0 - FastAPI 工程底座]]
所属分类：[[后端工程]]
相关位置：`app/main.py`

## 它解决什么问题

后端服务需要一个统一入口，让服务器知道应该加载哪个 FastAPI 应用。没有清晰入口时，后续路由、配置和启动逻辑容易混在一起。

## 在本项目中怎么用

`app/main.py` 中定义了 `create_app()`，它负责：

- 读取配置。
- 创建 `FastAPI` 实例。
- 注册 `health_router`。
- 暴露 `app` 给 `uvicorn` 加载。

## 为什么这样设计

用 `create_app()` 可以把“创建应用”和“运行应用”分开。后续如果要在测试中创建不同配置的应用，或者在启动时注册数据库、日志、异常处理，都可以扩展这个函数。

## 面试可能怎么问

为什么不直接在 `main.py` 里写所有接口？

## 你应该怎么回答

因为项目后续会有文档导入、搜索、问答、来源管理等多个模块。如果所有接口都写在 `main.py`，模块边界会很快失控。我把 `main.py` 作为应用组装入口，把具体接口放到独立 router 文件里，后续扩展更清晰，也更容易测试。
