# 阶段分支开发

所属阶段：[[阶段 0 - FastAPI 工程底座]]
所属分类：[[项目方法论]]
相关位置：`AGENT.MD`

## 它解决什么问题

如果所有功能都在一个线程、一个分支里推进，上下文和代码变更会混在一起。阶段分支开发用于让每个阶段的目标、变更和验证结果清楚可追踪。

## 在本项目中怎么用

阶段 0 使用分支：

```text
codex/phase-0-health-api
```

后续阶段建议使用：

```text
codex/phase-1-document-ingestion
codex/phase-2-vector-search
codex/phase-3-cited-chat
```

## 为什么这样设计

这接近真实团队里的 milestone 或 feature branch 工作方式。每个阶段结束后，可以独立检查代码、文档、测试和知识库。

## 面试可能怎么问

你是怎么组织这个项目开发过程的？

## 你应该怎么回答

我按 RAG 系统演进路线拆成阶段，每个阶段一个主线程和一个分支。阶段结束时同步更新 README、progress、architecture 和 Obsidian 知识库。这样项目不只是能跑，也能复盘每一步为什么这样做。
