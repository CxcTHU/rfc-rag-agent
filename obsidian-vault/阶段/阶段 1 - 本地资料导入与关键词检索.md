# 阶段 1 - 本地资料导入与关键词检索

所属索引：[[阶段索引]]

## 阶段目标

打通第一条本地资料链路：导入 Markdown/TXT/PDF，清洗文本，切分 chunk，保存到 SQLite，并用关键词检索返回相关片段。

## 完成内容

- 建立 SQLAlchemy 数据库层，包含 `documents` 和 `chunks` 两张表。
- 实现 parser、cleaner、splitter，把 Markdown/TXT/PDF 读成文本、清洗并切成 chunk。
- 实现 repository 和 ingestion service，串起“读取文件 -> 清洗 -> 切分 -> 保存”的导入链路。
- 接入 FastAPI：
  - `POST /documents/import`
  - `GET /documents`
  - `GET /documents/{document_id}/chunks`
  - `POST /search`
- 实现阶段 1 关键词检索：
  - 中文/英文同义词扩展
  - 标题和路径加分
  - 领域泛词降权
  - `metadata_record` 来源均衡
- 建立堆石混凝土种子资料、题录元数据语料库和来源目录。
- 建立关键词检索评测集 `data/evaluation/keyword_queries.csv` 和评测脚本 `scripts/evaluate_keyword_search.py`。

## 验证结果

- `python -m pytest`：38 个测试通过。
- `scripts/evaluate_keyword_search.py`：15/15 个代表性查询通过。
- `metadata_record` 在评测 top 结果中的最高比例控制在 0.50。

## 面试表达

阶段 1 我先完成了 RAG 系统的数据入口和关键词检索 baseline，而不是直接接大模型。

我把资料导入拆成 parser、cleaner、splitter、repository 和 ingestion service，每一步职责单一、可以单独测试。关键词检索先提供一个可解释的检索基线，再通过评测集校准中文、英文、同义词、标题加分和来源均衡。这样阶段 2 接入 embedding 向量检索时，就有一个明确的对照标准。
