1. `app/api/agent.py`：同步/SSE 入口、短路响应与持久化。
2. `app/services/agent/tool_calling_service.py`：薄服务门面。

## 恢复顺序

1. 先读 `AGENT.MD`、`docs/progress.md`、根 `task_plan.md/findings.md/progress.md/handoff.md`。
2. 确认分支、工作树与 staged diff，没有进入原工作区无关文件。
3. 读 Phase 68 四个阶段文件和 README/architecture 顶部单链路说明。
4. 从 API 沿 `ToolCallingAgentService -> RunCoordinator -> ToolExecutor -> registry -> AgentToolbox`
   复核引用，不因文件名误删 `legacy_toolbox.py`。
5. fresh 跑所有门禁；任何失败先修复或记录 blocker，禁止提交失败状态。
6. 需要回滚时使用 Git revision 和依赖恢复，不新增在线 legacy mode。
